Chapter 5: Programming Algorithms & Software Testing

Phases of Computer program development:

Phase 1 Understand the Problem
Phase 2 Design an algorithm to solve the problem
Phase 3 Write the program code in appropriate language based on the algorithm
Phase 4 Enter the program and save
Phase 5 Execute the program
Phase 6 Test the program, find errors, debug, then restart
Phase 7 Document and evaluate the program

Programming Structures:

Sequential structure: Top-down execution
Decision structures: if/then/else
Interative: while loop: continue looping as long something is true repeat/until loop: continue looping until something is true for loop: loop a given number of times
Recursive structures: Repeat a set of instructions as a subtask of itself Repetition continues until: degenerative case, which is a termination condition, is met (degenerative case is also called a "base case")
Two classic programmng/database problems Sorting: Descending order/Ascending order Sorting numbers Sorting strings (in accordance with ASCII values) Searching: find specific data that meet a certain criteria In database: Filters: temporary search Queries: search criteria are saved so they don't have to be recreated Two popular search techniques: Sequential search: search from top to bottom or bottom to top, one item at a time Binary search: only works if data is SORTED first start in the middle, then go to half of that, etc.
Two Classic program goals Efficiency and correctness

Software testing and verification Kinds of Testing:

Acceptance testing A test to show what quality a data base is set at
Integration Testing This is a test that interacts with the database
Unit testing This is when each part of the unit is exposed to differenct tests
Functional Testing This is when the unit is used and tested for durability and function
Regression Testing Testing to insure no update or addition has broke any existing functionality
Stress testing Testing the durability of the unit
Usability Testing Testing new work on the unit
Software testing is done to detect: Architectural flaws Poor design decisions Invalid or incorrect functionality Security vulnerabilities Scalability issues
Testing best practices: Continous testing Configuration management Servic virtualization Defect or "bug tracking" Metrics and reporting
